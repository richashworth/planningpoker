<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> POST /setLabel | Untrusted label text from browser |
| WebSocket broadcast -> all clients | Label content rendered in all participant browsers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | POST /setLabel | mitigate | Validate caller is host via `sessionManager.getHost(sessionId)` check — same pattern as kick/promote |
| T-08-02 | Tampering | Label content | mitigate | Server-side: trim to 100 chars, strip control characters. Client-side: React auto-escapes text in JSX so no XSS risk from label rendering |
| T-08-03 | Denial of Service | POST /setLabel | accept | Rate limiting not needed at ASVS L1; label is small string, same session validation as other endpoints. Debounced client-side (300ms) reduces call volume |
| T-08-04 | Information Disclosure | Label in RESULTS_MESSAGE | accept | Labels are visible to all session members by design; no sensitive data expected. Session membership required to subscribe to WebSocket topic |
</threat_model>
