<objective>
Wire the enriched WebSocket users payload ({users, host}) into Redux state and display a visual host indicator next to the host's name in the participants list.

Purpose: Phase 5 enriched the WebSocket users message with a `host` field, but the frontend still treats the payload as a plain array. This plan parses the enriched payload, stores host identity in Redux, and renders a host badge so all participants can identify the current host (HOST-03). Non-host users see the indicator but no action controls (UI-02 — controls come in Plan 02).

Output: Updated reducers, action creator, and UsersTable with host star icon.
</objective>

```js
case USERS_UPDATED:
  return action.payload.users || action.payload;
```

The fallback `|| action.payload` provides backwards compatibility if payload is already an array (defensive).
