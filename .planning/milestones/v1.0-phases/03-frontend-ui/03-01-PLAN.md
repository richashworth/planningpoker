# Plan 03-01: Scheme Selector on CreateGame Page

<objective>
Add a scheme selector (ToggleButtonGroup), custom values input (TextField), and meta-card toggles (Switch) to the CreateGame page. Wire the selected scheme params through the createGame action creator to the backend API.
</objective>

## must_haves
- ToggleButtonGroup with 4 options: Fibonacci (default), T-shirt, Simple, Custom
- Custom values TextField appears only when "Custom" is selected
- Inline validation on custom values (2-20 values, max 10 chars, duplicates flagged)
- Two Switch toggles for ? (unsure) and Coffee (break), both defaulting to on
- createGame action sends selected scheme params to backend

```js

  includeCoffee: schemeOptions.includeCoffee

Add local state variables after the existing `playerName` state:

const [schemeType, setSchemeType] = useState('fibonacci')
const [customValues, setCustomValues] = useState('')

const [includeCoffee, setIncludeCoffee] = useState(true)

```

Add a validation function:
```js
const validateCustomValues = (input) => {
  const values = input.split(',').map(v => v.trim()).filter(v => v.length > 0)
  const unique = [...new Set(values)]
  if (unique.length < 2) return 'At least 2 values required'
  if (unique.length > 20) return 'At most 20 values allowed'
  const tooLong = unique.find(v => v.length > 10)
  if (tooLong) return `Value "${tooLong}" exceeds 10 characters`
  if (unique.length < values.length) return 'Duplicate values removed'
  return ''
}

const handleCustomChange = (e) => {
  const val = e.target.value
  setCustomValues(val)
  if (val.trim()) {
    setCustomError(validateCustomValues(val))
  } else {
    setCustomError('')
  }
}
```

Add a submit guard - the "Duplicate values removed" message is informational (not blocking), so only block submission when values are truly invalid:
```js
const isCustomValid = () => {
  if (schemeType !== 'custom') return true
  const values = customValues.split(',').map(v => v.trim()).filter(v => v.length > 0)
  const unique = [...new Set(values)]
  if (unique.length < 2 || unique.length > 20) return false
  if (unique.find(v => v.length > 10)) return false
  return true
}
```

Update handleSubmit to pass scheme options:
```js
const handleSubmit = (e) => {
  e.preventDefault()
  const nameRegex = /^[a-zA-Z0-9 _-]{3,20}$/
  if (!nameRegex.test(playerName)) return
  if (!isCustomValid()) return
  dispatch(createGame(playerName, {
    schemeType,
    customValues: schemeType === 'custom' ? customValues : null,
    includeUnsure,
    includeCoffee
  }, () => {
    dispatch(gameCreated())
    navigate('/game')
  }))
}
```

Add the scheme selector JSX between the NameInput and Submit button. The full form body should be:

```jsx

  />
</Box>

<Button type="submit" variant="contained" fullWidth size="large" disableElevation disabled={!isCustomValid()}>
  Start Game
</Button>
```
