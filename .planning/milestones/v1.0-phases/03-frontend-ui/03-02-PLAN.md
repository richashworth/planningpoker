# Plan 03-02: Dynamic Vote Cards and Scheme-Aware Results Chart

<objective>
Replace hardcoded LEGAL_ESTIMATES usage in Vote.jsx and ResultsChart.jsx with dynamic legalEstimates from Redux state, so vote cards and chart labels reflect the session's actual estimation scheme.
</objective>

## must_haves
- Vote cards render from Redux `state.game.legalEstimates` instead of hardcoded LEGAL_ESTIMATES
- Results chart labels and aggregation use Redux `state.game.legalEstimates`
- No hardcoded LEGAL_ESTIMATES import in Vote.jsx or ResultsChart.jsx

## Tasks

<task id="1">
<title>Make Vote.jsx use dynamic legalEstimates from Redux</title>
<read_first>
- planningpoker-web/src/containers/Vote.jsx
- planningpoker-web/src/reducers/reducer_game.js
- planningpoker-web/src/config/Constants.js
</read_first>
<action>
In Vote.jsx:

1. Remove the Constants.js import line:
```js
// REMOVE: import { COFFEE_SYMBOL, LEGAL_ESTIMATES } from '../config/Constants';
```

2. Add a Redux selector for legalEstimates after the existing useSelector calls:
```js
const legalEstimates = useSelector(state => state.game.legalEstimates);
```

3. Replace line 59:
```js
// BEFORE:
const allValues = [...LEGAL_ESTIMATES, COFFEE_SYMBOL];

// AFTER:
const allValues = legalEstimates;
```

4. Remove the special coffee symbol font-size conditional from the card rendering. Currently line 84 has:
```js
...(val === COFFEE_SYMBOL && { fontSize: '1.5rem' }),
```
Since COFFEE_SYMBOL is no longer imported, replace with a Unicode check:
```js
...(val === '\u2615' && { fontSize: '1.5rem' }),
```
</action>
<acceptance_criteria>

</acceptance_criteria>
</task>
