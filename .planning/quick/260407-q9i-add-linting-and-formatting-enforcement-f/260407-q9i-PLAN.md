  <action>
Install dev dependencies in planningpoker-web:
```
npm install --save-dev eslint@^8 eslint-plugin-react@^7 eslint-plugin-react-hooks@^4 prettier eslint-config-prettier
```
Use ESLint 8 (not 9) — flat config is not needed and ESLint 8 has broader plugin compatibility for plain JS/JSX projects.

Create `planningpoker-web/.eslintrc.cjs`:
```js
module.exports = {
  root: true,
  env: {
    browser: true,
    es2022: true,
  },
  extends: [
    'eslint:recommended',
    'plugin:react/recommended',
    'plugin:react/jsx-runtime',
    'plugin:react-hooks/recommended',
    'prettier',
  ],
  parserOptions: {
    ecmaVersion: 'latest',
    sourceType: 'module',
    ecmaFeatures: { jsx: true },
  },
  settings: {
    react: { version: 'detect' },
  },
  rules: {
    'react/prop-types': 'off',
    'no-unused-vars': ['warn', { argsIgnorePattern: '^_', varsIgnorePattern: '^_' }],
    'no-console': 'off',
  },
  ignorePatterns: ['build/', 'dist/', 'node_modules/'],
}
```

Key points:
- `eslint-config-prettier` last in extends to disable formatting rules that conflict with Prettier.
- `react/prop-types` off — project doesn't use PropTypes.
- `no-unused-vars` as warn (not error) to avoid breaking on existing code patterns.
- `no-console` off — project uses console.error intentionally.
- Existing `eslint-disable-next-line` comments will work fine (ESLint supports them natively).

Create `planningpoker-web/.prettierrc`:
```json
{
  "semi": false,
  "singleQuote": true,
  "jsxSingleQuote": false,
  "tabWidth": 2,
  "trailingComma": "all",
  "printWidth": 100,
  "bracketSpacing": true
}
```
This matches observed style: no semicolons, single quotes, JSX double quotes, 2-space indent.

Create `planningpoker-web/.prettierignore`:
```
build/
dist/
node_modules/
*.html
```

Add npm scripts to planningpoker-web/package.json:
```json
"lint": "eslint src/ --ext .js,.jsx",
"lint:fix": "eslint src/ --ext .js,.jsx --fix",
"format": "prettier --write \"src/**/*.{js,jsx,json,css}\"",
"format:check": "prettier --check \"src/**/*.{js,jsx,json,css}\""
```

After config is in place, run `npm run format` to auto-format all files to match Prettier config, then run `npm run lint` and fix any remaining lint errors. The goal is zero lint errors and zero Prettier diffs on the existing codebase. If any ESLint rules produce errors on existing code that cannot be auto-fixed, either add inline disable comments or adjust the rule to warn level — do NOT leave errors.

  <name>Task 2: Configure Spotless for backend Java formatting</name>
  <files>
    planningpoker-api/build.gradle
  </files>
  <action>
Add the Spotless plugin to planningpoker-api/build.gradle:

In the plugins block, add:
```groovy
id 'com.diffplug.spotless' version '7.0.2'
```

After the plugins block, add Spotless configuration:
```groovy
spotless {
    java {
        target 'src/main/java/**/*.java', 'src/test/java/**/*.java'
        googleJavaFormat('1.22.0')
        removeUnusedImports()
        trimTrailingWhitespace()
        endWithNewline()
    }
}
```

Run `./gradlew spotlessApply` to format all existing Java files. Then run `./gradlew spotlessCheck` to confirm zero violations remain.

If Google Java Format changes indentation or brace style from what currently exists, that is expected and acceptable — the point is to pick a standard and enforce it going forward.

Add a dependency so `check` includes spotless:
```groovy
tasks.named('check') {
    dependsOn tasks.named('jacocoTestReport')
    dependsOn tasks.named('spotlessCheck')
}
```

  <name>Task 3: Add CI lint job gating unit-tests and e2e-tests</name>
  <files>
    .github/workflows/ci.yml
  </files>
  <action>
Add a new `lint` job to `.github/workflows/ci.yml` that runs BEFORE unit-tests and e2e-tests.

The `lint` job:
```yaml
lint:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4

    - uses: actions/setup-java@v4
      with:
        distribution: temurin
        java-version: 21

    - uses: actions/setup-node@v4
      with:
        node-version: 22
        cache: npm
        cache-dependency-path: planningpoker-web/package-lock.json

    - name: Cache Gradle
      uses: actions/cache@v4
      with:
        path: |
          ~/.gradle/caches
          ~/.gradle/wrapper
        key: gradle-${{ hashFiles('**/*.gradle*', 'gradle/wrapper/gradle-wrapper.properties') }}
        restore-keys: gradle-

    - name: Install frontend dependencies
      run: cd planningpoker-web && npm ci

    - name: Frontend lint + format check
      run: cd planningpoker-web && npm run lint && npm run format:check

    - name: Backend format check
      run: ./gradlew spotlessCheck --no-daemon
```

Add `needs: lint` to both `unit-tests` and `e2e-tests` jobs so they only run after lint passes:
```yaml
unit-tests:
  needs: lint
  runs-on: ubuntu-latest
  ...

e2e-tests:
  needs: lint
  runs-on: ubuntu-latest
  ...
```

Install husky and lint-staged as dev dependencies in planningpoker-web:
```
cd planningpoker-web && npm install --save-dev husky lint-staged
```

Initialize husky from the project root (husky needs to be at the git root level):
```
cd /Users/richard/Projects/planningpoker
npx husky init
```

This creates `.husky/` directory at the repo root. Edit `.husky/pre-commit` to contain:
```sh
cd planningpoker-web && npx lint-staged

cd ..
# Run spotless on staged Java files
STAGED_JAVA=$(git diff --cached --name-only --diff-filter=ACM -- '*.java')
if [ -n "$STAGED_JAVA" ]; then
  ./gradlew spotlessApply --no-daemon
  echo "$STAGED_JAVA" | xargs git add
fi
```

Add lint-staged config to planningpoker-web/package.json:
```json
"lint-staged": {
  "src/**/*.{js,jsx}": [
    "eslint --fix",
    "prettier --write"
  ]
}
```

Note: husky v9 uses the `.husky/pre-commit` file directly (no `npx husky add` needed). The `prepare` script should be added to the ROOT package.json if one exists, or to planningpoker-web/package.json otherwise. Since this is a Gradle project without a root package.json, add a minimal root package.json if needed, OR configure husky via planningpoker-web's prepare script with a `cd ..` prefix. The simplest approach: add `"prepare": "cd .. && husky"` to planningpoker-web/package.json scripts (since npm install runs from planningpoker-web/).

Test the hook by staging a JS file and verifying lint-staged runs:
```
cd planningpoker-web && echo "// test" >> src/config/Constants.js && git add src/config/Constants.js && git commit --dry-run
```
Then revert the test change.
