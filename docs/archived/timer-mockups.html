Round Timer — Placement & Interaction Mockups

Seven options for adding a round timer feature (inspired by Miro, Teams, Parabol, Scrum Poker Online). Core interactions across all options: host chooses duration → start → countdown ticks → warn at 10s → expire. Host-only controls (start / pause / reset / extend); non-host sees read-only countdown. Ordered roughly from most subtle to most prominent. Pick one (or a hybrid).

A — Header chip

Tiny pill lives in the top app bar, next to the session ID chip.

SUBTLE · AMBIENT

Always-visible pill in the header

Highest screen-real-estate efficiency. The timer is present without ever occupying space inside the voting area. Host clicks the pill to open a menu (start/pause/duration). Feels like a "status bar" element.

Cross-cutting decisions (apply to any option)

  • Presets — 30s / 1m / 2m / 5m / custom. Remember last used in localStorage. Default probably 2m.
  • Auto-reveal on expiry? Setting per session (checkbox in duration picker). Some teams want strict "no more voting", some want a soft nudge.
  • State sync — needs backend support. Store startedAt + durationSeconds per session; broadcast on start/pause/reset over STOMP. Clients tick locally between updates (don't flood the wire).
  • Host-only controls — players see read-only countdown. Respect the existing admin/player split.
  • Audio cue — optional soft "ding" at 10s and 0s. Off by default, toggle in header menu.
  • Visual urgency — final 10 seconds should warn (amber), final 3 should pulse (red). Consistent across all options.
  • Pause vs. hard-stop — pause should exist (interruptions happen). Reset clears and returns to idle.
