Planning Poker — Layout Options

Two placement problems to solve. (1) The round-label Set button currently sits as an inline endAdornment inside the TextField — a textbox-with-trailing-action pattern that feels cramped and duplicates what Enter already does. (2) Export CSV competes with Next Item in the Results header, even though it's a low-frequency session-level action, not a round-level one. Below are 5 options grounded in common UX patterns (Material Design action hierarchy, NN/g "Save Buttons" guidance, Linear/Notion/GitHub inline editing). Each mockup shows the Vote screen on the left and the Results screen on the right.

Option A

Inline-edit heading + kebab overflow menu

Treat the round label as a title, not a form field. Click-to-edit heading, saves on Enter/blur — no button needed. Export CSV moves into a 3-dot overflow menu next to "Next Item", matching the Material "more actions" pattern. Primary action stays visually dominant.

Option B

Auto-save label + footer toolbar for secondary actions

Label saves automatically on blur or 1s after typing stops — "Set" button disappears entirely, replaced by a subtle "Saved" indicator. Export CSV (plus future exports) moves to a footer toolbar below the results chart, separated from the primary "Next Item" CTA. Common in Google Docs, Figma, Notion.

Option C

Icon-only buttons (compact toolbar)

Both secondary actions collapse to icon buttons with tooltips. Label TextField gets a small checkmark icon on the right; Results header gets a download-icon button instead of a labelled outlined button. Primary "Next Item" retains full label to remain dominant. Reduces visual weight by ~60% but costs discoverability.

Option D

Session toolbar above GamePane (actions move up one level)

Export CSV is a session-level action — it exports all rounds including ones you can't see on the current Vote/Results view. Moving it to a persistent toolbar above the game pane makes that scope explicit and keeps round-level UI focused on the active item. Label keeps auto-save (no button). "Next Item" remains the only action in the Results header.

Option E

Split-button primary + editable title with pencil

Keep actions in the Results header but merge them into a split button: the main surface advances to the next item; the chevron opens a menu with "Next Item + Export CSV", "Export only", etc. Label becomes a large editable title with a pencil affordance (click or hover to reveal). Progressive disclosure for secondary actions, single dominant CTA.

Trade-off summary

Option Label UX Export visibility Implementation effort Best for
A — Inline heading + kebab click-to-edit 1 click hidden medium Polished look, room for future secondary actions (copy, share, reset)
B — Auto-save + footer toolbar invisible visible, separated low Minimum change, cleanest action hierarchy
C — Icon-only buttons checkmark icon icon, no label low Compact; costs discoverability — tooltips mandatory
D — Session toolbar above pane invisible persistent, scoped medium Correct semantic scope; adds vertical space cost
E — Split button + title click-to-edit 2 clicks hidden higher Strong single CTA; split buttons have lower discoverability
F — Prominent card + history section invisible co-located w/ data medium Best information scent; largest visual change

My pick: B for minimum-diff shipping today, D if you're willing to touch layout above the pane. Both eliminate the Set button, both give Export CSV room to breathe without competing with Next Item. A is the nicest-looking but requires writing a new inline-edit component.
